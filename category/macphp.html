mac下php相关配置文件

Wed 16 January 2013
By 向远

In mac,php.

昨天偶尔想用php写一个发http请求来写一个api。
结果HttpRequest类找不到。

找资料时就发现有提及php.ini这个文件，由于之前也没有接触过这些东西。

sudo find / php.ini

命令查找了一个居然没有这个文件。

php -i | less

用上述命令查看下.

Configuration File (php.ini) Path   /etc
Loaded Configuration File   （none）
Scan this dir for additional .ini files (none)
Additional .ini files parsed    (none)

发现这个文件需要自己却创建一下.

cp php.ini.default php.ini

php.ini.default文件在/etc/目录下面，是一个默认的配置文件。需要把里面的display_error打开，否刚就不会出现错误信息在开发过程中。

回到最开始的问题，类找不到需要添加扩展。
phpinfo里面就可以发现下面这一项。
extension_dir   /usr/lib/php/extensions/no-debug-non-zts-20090626   /usr/lib/php/extensions/no-debug-non-zts-20090626

只xdebug.so扩展，添添加扩展。

在安装扩展时有一个命令pear
发现我的mac居然没有这个管理工具
/usr/lib/php/
在上面目录下面install-pear-nozlib.phar 这个文件
运行下面命令就可以安装成功了
php install-pear-nozlib.phar
